# Сервер раздачи статики - решение тестового задания

Моё решение задания: плейбук и роли Ansible, которые настраивают
сервер, отдающий картинки через nginx по пути `/images`.Согласно задания в качестве тестового хоста использую Docker-контейнер на Ubuntu 24.04.

Ниже коротко рассказываю почему сделал именно так.

## Команды

```bash
# 1. Генерирую ключ, которым Ansible будет ходить в контейнер.
#    Скрипт идемпотентный - если ключ уже есть, ничего не делает.
./scripts/gen_keys.sh

# 2. Поднимаю тестовое окружение (проброшены порты 2222->22 и 8080->80).
docker compose -f docker/docker-compose.yml up -d --build

# 3. Проверяю, что хост доступен.
ansible -i inventory/hosts.ini all -m ping

# 4. Накатываю плейбук.
ansible-playbook -i inventory/hosts.ini site.yml
```

После этого открываю результат:

- список файлов - http://localhost:8080/images/
- отдельная картинка - `http://localhost:8080/images/<имя_файла>`

Зайти на хост по SSH:

```bash
ssh -i keys/id_ansible -p 2222 deploy@localhost
```

Перед запуском нужен Ansible (core ≥ 2.15) и коллекция `ansible.posix`
(она даёт модуль `authorized_key`):

```bash
ansible-galaxy collection install -r requirements.yml
```

## Что внутри

Плейбук `site.yml` прогоняет роли в таком порядке:

| Роль     | Что делает                                                                                   |
| -------- | -------------------------------------------------------------------------------------------- |
| `common` | `apt update`/`upgrade` и ставит утилиты (`htop`, `ncdu`, `git`, `nano`) - список расширяемый |
| `users`  | Создаёт/удаляет пользователей и кастомные группы по описанию из vars                         |
| `zsh`    | Ставит `zsh` + `oh-my-zsh` тем пользователям, у кого shell - zsh                             |
| `ssh`    | Хардит `sshd`: без root-логина, без пустых паролей, `LogLevel VERBOSE`, без X11              |
| `nginx`  | Ставит nginx и поднимает vhost с `/images` (autoindex, gzip, логи, кэш на час)               |
| `static` | Скачивает архив с картинками на сервер и распаковывает в web-root                            |

## Решения, которые я принял по ходу

**Почему отдельный пользователь `deploy`.** Ansible подключается к контейнеру не
под root, а под специально заведённым пользователем `deploy` (создаётся в
Dockerfile, лежит ключ и беспарольный sudo). Это удобно тем, что роль `ssh`
спокойно отключает root-логин - мы всё равно ходим под `deploy` и связь не
теряем. Заодно это ближе к тому, как делается на реальных серверах.

**Почему в контейнере нет systemd.** Гонять systemd внутри Docker - лишняя боль
(нужен privileged, cgroups и т.д.). Поэтому sshd я стартую из `entrypoint.sh` и
держу контейнер живым через `sleep infinity` как PID 1. Важный момент: sshd
специально НЕ запускаю как PID 1, иначе хендлер «перезапустить ssh» из роли
убил бы контейнер. А nginx Ansible поднимает через обычный init-скрипт
`/etc/init.d/nginx` - он приезжает вместе с пакетом nginx и прекрасно работает
без systemd.

**Про gzip.** В стоковом `nginx.conf` уже есть `gzip on;` на уровне `http`,
поэтому отдельный файл в `conf.d` с тем же `gzip on;` ломал конфиг дублем
директивы. Я перенёс настройки gzip прямо в `server`-блок vhost - там они
переопределяют http-уровень без конфликта. Сжимаю только текстовые типы (svg,
css, js, json), jpeg/png/gif и так уже сжаты, гонять их через gzip смысла нет.
В подборке статики как раз SVG, так что сжатие реально срабатывает.

**Про идемпотентность.** Старался везде использовать штатные модули, которые
сами по себе идемпотентны (`user`, `group`, `apt`, `template` и т.д.).
В паре мест, где идемпотентность не «из коробки», поставил защиту: клон
oh-my-zsh закрываю через `creates`, а скачивание и распаковку статики гейчу по
файлу-маркеру (`stat` проверяет `.extracted`, и весь блок выполняется только на
первом прогоне). Это заодно позволяет удалять скачанный архив после распаковки,
не вызывая повторных загрузок. Проверял - второй прогон плейбука даёт
`changed=0`.

**Про статику с Google Drive.** Роль `static` качает архив прямо на сервере
(`get_url`) и распаковывает (`unarchive`). Для больших файлов Google Drive
показывает страницу-предупреждение вместо файла, поэтому в URL добавил
`confirm=t`. На случай если Drive всё-таки отдаст HTML вместо архива, я добавил
проверку MIME-типа с понятной ошибкой - чтобы сразу было видно, что пошло не
так, а не ловить кривую распаковку.

**Про переменные и defaults.** Старался нигде не хардкодить значения прямо в
тасках и шаблонах всё, что в принципе можно захотеть поменять, вынес в
переменные. Разложил их по двум уровням:

- У **каждой роли** есть свой `defaults/main.yml` с разумными значениями по
  умолчанию. За счёт этого роль самодостаточна: её можно взять и запустить отдельно,
  ничего дополнительно не передавая, и она отработает на дефолтах.

- В `group_vars/all/` лежит только то, что реально настраивается под конкретный
  деплой: список пользователей и групп, набор утилит, параметры хардинга sshd.
  Эти значения перекрывают defaults роли, поэтому в group_vars держу не «копию
  всего подряд», а именно данные деплоя - то, что задание просит задавать через
  vars-файлы.

Что это даёт на практике:
- **ТЗ выполняется буквально** - состояние пользователей, кастомные группы,
  список утилит и т.д. меняются правкой одного vars-файла, без захода в роли.
- **Список утилит легко расширяем** - требование задания: дописал
  строку в `packages.yml`, и всё
- **Ничего не дублируется.** Например, gzip-типы, web-root, тема oh-my-zsh,
  параметры sshd живут в одном месте.
- **Менять поведение можно без правки кода.** Чтобы, скажем, поднять кэш с часа
  до суток или сменить `LogLevel`, достаточно переопределить переменную - таски
  и шаблоны не трогаю.

## Где что настраивается

- **`group_vars/all/users.yml`** - кастомные группы и пользователи. У каждого
  можно задать имя, shell, состояние (`present`/`absent`) и т.д.
- **`group_vars/all/packages.yml`** - список утилит (просто дописываю в него
  нужное) и флаг апгрейда пакетов.
- **`group_vars/all/ssh.yml`** - параметры хардинга sshd вынесены в переменные,
  чтобы не лазить в шаблон.
- Остальные настройки (nginx, источник статики, тема oh-my-zsh) лежат в
  `defaults/main.yml` соответствующих ролей - там же их при желании можно
  переопределить.

## Что бы я доработал для production

Если бы это был реальный прод, в первую очередь добавил бы:

- **Секреты через `ansible-vault`.** Хеши паролей пользователей сейчас лежат в
  открытом vars-файле как демо-данные - в проде их место в зашифрованном
  vault-файле.
- **Больше хардинга sshd.** ТЗ перечисляет 4 директивы - их и сделал. Сверх того
  напрашиваются `MaxAuthTries`, `ClientAliveInterval`, `AllowGroups`. Полное
  `PasswordAuthentication no` тут сознательно не ставил: задание требует
  заводить пользователям пароли, так что отключать парольный вход целиком было
  бы противоречиво - это решение уровня политики конкретного окружения.
- **Firewall** (ufw/nftables) - оставить открытыми только 22 и 80/443.
- **TLS** - задание про 80 порт и HTTP, но в проде это был бы nginx + certbot и
  редирект на HTTPS.
- **CI** - прогон `ansible-lint`/`yamllint` и Molecule-сценарий (converge +
  idempotence + verify) в GitHub Actions на каждый PR.
- **Источник статики** - сейчас это Google Drive из задания (URL вынесен в
  переменную, есть проверка и ручной фолбэк). В проде источником был бы
  artifact-репозиторий или S3, а не Drive.

## Ручной фолбэк для статики

Если автоскачивание с Google Drive по какой-то причине не отработало, можно
просто закинуть архив руками:

```bash
docker cp ./images.zip static-server:/tmp/static_images_archive
docker exec static-server bash -c \
  'unzip -o /tmp/static_images_archive -d /var/www/static/images && \
   chown -R www-data:www-data /var/www/static/images && \
   touch /var/www/static/images/.extracted'
```

## Проверка 

```bash
# Идемпотентность - второй прогон должен быть changed=0.
ansible-playbook -i inventory/hosts.ini site.yml

# Раздача статики.
curl http://localhost:8080/images/            # листинг каталога
curl -I http://localhost:8080/images/<файл>   # 200 + Expires/Cache-Control

# Хардинг ssh.
docker exec static-server sshd -T | grep -Ei 'permitrootlogin|permitemptypasswords|loglevel|x11forwarding'

# Линтеры.
ansible-lint
yamllint .
```

Прогонял `ansible-lint` на профиле `production` и `yamllint` - оба проходят
без замечаний.

docker compose -f docker/docker-compose.yml down
